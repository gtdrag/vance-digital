**src/db/queries/budget.ts** — Budget query functions:
- `getMonthlyTotal(db)`: SUM(amount_usd) from transactions WHERE created_at >= first day of current calendar month (ISO string comparison). Return 0 if null. Calendar month per research open question recommendation.
- `getSpendingByCategory(db)`: GROUP BY category, SUM(amount_usd) for current calendar month. Return Record<TransactionCategory, number> with 0 defaults for categories with no transactions.
- `getTransactionHistory(db, { limit?: number })`: Recent transactions ordered by created_at DESC, default limit 20.
- `insertTransaction(db, tx)`: INSERT into transactions table, return inserted row.

**src/budget/tracker.ts** — Budget enforcement:
- `canSpend(db, config, amount)`: Load monthly total, check if total + amount <= config.budget.monthlyCapUsd. Return `{ allowed: boolean; currentSpend: number; remaining: number; reason?: string }`.
- `recordTransaction(db, { category, amountUsd, description?, referenceId? })`: Insert transaction via queries/budget. Return inserted transaction.
- `getMonthlySpending(db)`: Delegate to getMonthlyTotal.
- `getBudgetStatus(db, config)`: Calculate spent, remaining, percentUsed. Set `isWarning` if percentUsed >= 80, `isExceeded` if percentUsed >= 100.
- `getSpendingBreakdown(db)`: Delegate to getSpendingByCategory, format for display.

<task type="auto" tdd="true">
  <name>Task 2: Budget projections — renewals and Next 30 days forecast (TDD)</name>
  <files>src/budget/projections.ts, src/budget/projections.test.ts</files>
  <behavior>
    - getUpcomingRenewals(db, days=30): returns domains with renewal_date within next N days
    - getUpcomingRenewals(db): returns empty array when no domains exist
    - getUpcomingRenewals(db): excludes domains with renewal_date in the past
    - getUpcomingRenewals(db): excludes domains with status='failed'
    - getRenewalProjection(db, days=30): returns total projected renewal cost (sum of renewal_cost_usd)
    - estimateApiCosts(db, days=30): estimates next-30-day API costs based on recent 30-day average
    - estimateApiCosts(db): returns 0 when no API transactions exist
    - getNext30DaysProjection(db, config): returns { renewalCost, estimatedApiCost, totalProjected, remainingBudget, willExceedCap }
    - getNext30DaysProjection correctly identifies when projected costs will exceed monthly cap
  </behavior>
  <action>
**src/budget/projections.ts** — Projection calculations:
- `getUpcomingRenewals(db, days = 30)`: Query domains table WHERE renewal_date BETWEEN now AND now+days, status != 'failed'. Return array of { name, renewalDate, renewalCostUsd }.
- `getRenewalProjection(db, days = 30)`: SUM(renewal_cost_usd) of upcoming renewals. Return number.
- `estimateApiCosts(db, days = 30)`: Look at API transactions (categories starting with 'api_') from past 30 days, calculate daily average, multiply by projection days. Return estimated cost. If no history, return 0.
- `getNext30DaysProjection(db, config)`: Combine renewal projection + API cost estimate. Calculate:
  - `renewalCost`: from getRenewalProjection
  - `estimatedApiCost`: from estimateApiCosts
  - `totalProjected`: renewalCost + estimatedApiCost
  - `remainingBudget`: config.budget.monthlyCapUsd - getMonthlySpending(db)
  - `willExceedCap`: totalProjected > remainingBudget
  Return typed projection object.

- Test API cost estimation: insert 10 API transactions over past 30 days, verify estimate extrapolates correctly
